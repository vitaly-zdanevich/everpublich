Everpublich

Sync an Evernote notebook to a static Zola blog/website with RSS, tags, search, a calendar, podcast feed, media playback, expanded links-to-widgets, and S3/CloudFront hosting.

A better version of postach.io and notesrss.com.

Share your notebook read-only with share@everpublich.my, and we will publish and sync it.

Your address will be <your-notebook-name>.everpublich.my

If you stop sharing the notebook, the generated website will be deleted on the next sync.

Evernote no longer exposes native note tags in shared notebooks, so Everpublich reads #tags from note content instead.

What ships in the MVP

  • Hourly full regeneration
  • Zola static search enabled by default, with Google search as a setting
  • Images, audio, video, PDFs, 3D models, archive trees, text/subtitle previews, and attachments copied from Evernote and served from S3 through CloudFront Content Delivery Network for fast page loads
  • HEIC/HEIF, DNG, TIFF, BMP, JPEG 2000, JPEG XL, PSD, RAW, and other browser-hostile images are served as AVIF or extracted JPEG, so you can keep originals in Evernote
  • SVG is rendered directly; Adobe Illustrator, EPS, PS, and compressed SVG files get an AVIF preview plus the original file download
  • RSS, sitemap.xml, tag pages from note-content #tags, About page, calendar, and podcast.xml for notes with #podcast
  • Title commands: everpublich:about for About, everpublish:Page for pages, everpublich:#tag for nav links, and everpublich:config for options such as widgets: off plus CSS/JS code blocks
  • Standalone links can expand to media widgets, including Reddit posts and subreddit widgets
  • SEO-friendly output with minimal JavaScript, used only for static search, and max-Brotli text objects in S3
  • Internal Evernote links inside the shared notebook become relative website links
  • Optional GitHub backup repository with public/private visibility
  • Mobile-friendly design with true black dark mode via prefers-color-scheme
  • Offline support - right in a browser
